# harness-gitops-agent-bootstrap

This chart wires together two lifecycle actions in one release:

1. Creates a `HarnessGitopsAgent` custom resource (consumed by your controller).
2. Installs Harness `gitops-helm` (which deploys Argo CD + GitOps agent runtime).

## Security model

- The Harness PAT is **not** stored in chart values.
- You must pre-create `Secret/<apiKeySecretRef>` with key `api_key`.
- The runtime agent token is generated by the controller into `tokenSecretRef` and consumed by gitops-helm via `agent.existingSecrets.agentToken`.

## Install

1. Install controller first:

```bash
helm upgrade --install hgac chart/harness-gitops-agent-controller \
  -n harness-system --create-namespace
```

2. Create Harness API key secret in target namespace (example: `argocd-agent`):

```bash
kubectl create namespace argocd-agent --dry-run=client -o yaml | kubectl apply -f -
kubectl -n argocd-agent create secret generic harness-api-key-secret \
  --from-literal=api_key='<HARNESS_PAT>'
```

3. Build dependency and install bootstrap chart:

```bash
helm dependency update chart/harness-gitops-agent-bootstrap

helm upgrade --install hub-bootstrap chart/harness-gitops-agent-bootstrap \
  -n argocd-agent \
  --create-namespace \
  --set gitopsAgent.harness.identity.accountIdentifier=<ACCOUNT_ID> \
  --set gitopsAgent.harness.identity.orgIdentifier=<ORG_ID> \
  --set gitopsAgent.harness.identity.projectIdentifier=<PROJECT_ID> \
  --set gitopsAgent.harness.identity.agentIdentifier=<AGENT_IDENTIFIER> \
  --set harnessAgent.spec.apiKeySecretRef=harness-api-key-secret \
  --set harnessAgent.spec.tokenSecretRef=my-agent-token \
  --set gitopsAgent.agent.existingSecrets.agentToken=my-agent-token
```

## Notes

- Keep `harnessAgent.spec.tokenSecretRef` and `gitopsAgent.agent.existingSecrets.agentToken` identical.
- Do not use `--wait` on first install; the controller needs time to create the token secret.
