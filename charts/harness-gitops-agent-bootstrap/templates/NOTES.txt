This chart installs:
1. HarnessGitopsAgent custom resource for your controller
2. Harness gitops-helm chart for in-cluster agent runtime

Pre-requisites:
- The harness-gitops-agent-controller must already be installed and running
- Secret/{{ .Values.harnessAgent.spec.apiKeySecretRef }} must exist in namespace {{ .Release.Namespace }} with key `api_key`

Verify lifecycle:
1. Check CR status fields:
   kubectl get harnessgitopsagent -n {{ .Release.Namespace }} {{ default .Values.gitopsAgent.harness.identity.agentIdentifier .Values.harnessAgent.metadata.name }} -o yaml

2. Wait for token secret generated by controller:
   kubectl get secret -n {{ .Release.Namespace }} {{ .Values.harnessAgent.spec.tokenSecretRef }}

3. Check agent rollout:
   kubectl get deploy,pods -n {{ .Release.Namespace }}
   kubectl logs -n {{ .Release.Namespace }} deploy/{{ .Release.Name }}-gitops-agent -f
